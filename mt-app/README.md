Multi-Service Monitoring Platform
프로젝트 비전
개발자들이 여러 서비스를 각각 모니터링하는 번거로움을 해결하고, 하나의 통합 대시보드에서 모든 서비스를 효율적으로 관리할 수 있는 플랫폼을 제공합니다.

핵심 가치
1. 프라이버시 우선 (Privacy-First)
데이터 수집 없음: 사용자의 서비스 데이터를 외부로 전송하지 않습니다
로컬 저장: 모든 설정은 Docker Compose YAML 및 JSON 파일에 로컬 저장
개인정보 보호: 민감한 서비스 정보와 메트릭이 외부에 노출되지 않습니다
2. 셀프 호스팅 (Self-Hosted)
Docker Hub를 통해 배포되며, 사용자가 원하는 환경에 직접 설치 가능합니다:

로컬 개발 환경
홈서버
클라우드 인스턴스 (AWS, GCP, Azure 등)
3. 오픈소스 정신
투명성: 모든 코드가 공개되어 검증 가능
커뮤니티 주도: 사용자의 피드백과 기여를 환영
자유로운 커스터마이징: 각자의 요구사항에 맞게 수정 가능
주요 기능
통합 대시보드: 모든 서비스의 상태를 한눈에 확인
실시간 모니터링: 서비스 헬스, 레이턴시, 에러율 등 실시간 추적
알림 시스템: 중요한 이벤트 발생 시 즉각적인 알림
로그 관리: 통합 로그 뷰어 및 검색 기능
보안 대시보드: 보안 이벤트 및 위협 모니터링
설치 및 사용

# Docker Hub에서 이미지 Pull
docker pull [your-dockerhub-username]/mt-app:latest

# Docker Compose로 실행
docker-compose up -d
모든 설정은 docker-compose.yml 및 설정 JSON 파일에서 관리됩니다.

왜 이 프로젝트인가?
기존 모니터링 솔루션들의 문제점:

🔒 보안 우려: 민감한 서비스 데이터를 외부 SaaS에 전송
💰 비용: 서비스 수에 따라 증가하는 구독 비용
🔧 제한된 커스터마이징: 폐쇄적인 플랫폼 구조
우리의 솔루션:

✅ 완전한 데이터 통제: 모든 데이터가 사용자 환경에 보관
✅ 무료: 오픈소스로 제공
✅ 완전한 자유: 소스 코드 수정 및 확장 가능
기여하기
이 프로젝트는 오픈소스이며, 여러분의 기여를 환영합니다!
